## 分类详解

### 单轮路由器

适用于简单的查询-模型映射场景，单次路由决策。

#### KNN Router

**核心特点**：
- 无需训练，基于实例的学习
- 根据查询相似度选择历史最优模型
- 高度可解释（可查看相似查询）
- 支持增量学习（实时添加新样本）

**适用场景**：
- 小到中等规模数据集（10-10,000样本）
- 需要快速原型验证
- 数据分布经常变化
- 需要可解释的决策过程
- 计算资源有限

**最佳实践**：
```yaml
hparam:
  n_neighbors: 5
  weights: "distance"
  metric: "cosine"
  n_jobs: -1
```

---

#### MLP Router

**核心特点**：
- 多层感知器分类器
- 学习非线性查询-模型映射
- 推理速度快（单次前向传播）

**适用场景**：
- 中等规模数据集（1,000-100,000样本）
- 需要快速推理
- 查询分布相对稳定
- 有充足的训练数据

**最佳实践**：
```yaml
hparam:
  hidden_layer_sizes: [128, 64]
  activation: "relu"
  solver: "adam"
  alpha: 0.0001
```

---

#### SVM Router

**核心特点**：
- 支持向量机分类器
- 核技巧处理非线性边界
- 理论保证（最大化间隔）

**适用场景**：
- 中等规模数据集（100-10,000样本）
- 需要理论保证
- 高维特征空间

**最佳实践**：
```yaml
hparam:
  kernel: "rbf"
  C: 1.0
  gamma: "scale"
```

---

#### ELO Router

**核心特点**：
- 基于Elo评分系统的全局排序
- 查询无关（所有查询路由到同一模型）
- 无超参数
- 快速推理

**适用场景**：
- 需要全局模型排序
- 查询类型单一
- 作为基线对比

---

#### Automix Router

**核心特点**：
- 自验证机制
- 三种路由策略（阈值、POMDP、自一致性）
- 自动检测大小模型
- 理论保证（POMDP）

**适用场景**：
- 成本敏感应用
- 混合难度工作负载
- 有小模型和大模型

**最佳实践**：
```yaml
hparam:
  routing_method: "POMDP"
  num_bins: 8
  small_model_cost: 1
  large_model_cost: 50
```

---

### 多轮路由器

适用于复杂查询，支持分解、多步骤推理。

#### Router-R1

**核心特点**：
- Agentic多轮推理
- 动态查询路由池
- 迭代细化答案
- 完整推理追踪

**适用场景**：
- 复杂查询需要多步推理
- 需要可解释的决策过程
- 有高质量路由池API
- GPU资源充足

---

#### LLM Multi-Round Router

**核心特点**：
- LLM分解+路由
- 零样本无需训练
- 基于模型描述智能路由
- LLM聚合响应

**适用场景**：
- 无训练数据
- 复杂查询需要分解
- 模型能力描述清晰

---

### 个性化路由器

学习用户特定偏好，提供个性化路由。

#### GMTRouter

**核心特点**：
- 异构HeteroGNN
- 5种节点类型，21种边类型
- 多轮会话支持
- 成对偏好学习

**适用场景**：
- 个性化对话机器人
- 多用户平台
- 多轮对话场景

---

#### Personalized Router

**核心特点**：
- 用户节点集成
- 异构图（用户、查询、任务、LLM）
- 用户偏好学习
- 多任务支持

**适用场景**：
- 多用户平台
- 用户偏好各异
- 有用户交互历史

## 使用建议

### 按数据规模选择

#### 小数据集（<1,000样本）
- **首选**：KNN Router
- **原因**：无需训练，适合快速原型

#### 中等数据集（1,000-10,000样本）
- **首选**：MLP Router
- **原因**：平衡性能和训练成本

#### 大数据集（>10,000样本）
- **首选**：RouterDC
- **原因**：充分利用大规模数据

## 常见问题

### Q1: 如何选择第一个路由器？

**A**: 如果您刚开始使用LLMRouter：

1. **有训练数据**：从KNN Router开始
2. **无训练数据**：从LLM Multi-Round Router开始
3. **只需基线**：从ELO Router开始

### Q2: 什么时候应该升级路由器？

**A**: 考虑升级当：
- 精度不足：当前路由精度 < 80%
- 延迟过高：推理时间超过SLA
- 成本过高：API成本超出预算
- 数据增长：训练数据规模增加

### Q3: 如何评估路由器性能？

**A**: 关键指标：
1. 路由精度：正确路由的比例
2. 端到端性能：任务完成质量
3. 成本节省率：相比总是用大模型
4. 推理延迟：平均响应时间
